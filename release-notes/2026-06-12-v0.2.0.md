### Added
- Member portal redesign **phase 4 — Account & Support**:
  - **Account** is now a single two-column page — the profile form inline (the
    separate `/dashboard/account/edit` route redirects here) plus a
    Membership / Billing / Security aside (Change password → real flow).
  - **Business profiles**: members can own multiple businesses, each with contact
    details, five social handles (Instagram, Facebook, LinkedIn, TikTok, X), and a
    square logo upload; one is marked Primary. New `businesses` table + public
    `business-logos` bucket + `set_primary_business` RPC
    (`20260612200000_businesses.sql`).
  - **Support**: a concierge contact form (topic, message, **file attachments**
    up to 5) writing to a new `support_messages` table with a private
    `support-attachments` bucket, plus an FAQ accordion
    (`20260612190000_support.sql`).
- Member portal redesign **phase 3 — Benefits**: the Benefits page now reads the
  **live partner catalog** from Supabase and opens a **redemption drawer** per
  partner — promo **code** (copy), private **link** (copy), or **concierge**
  message — recording each via a `redeem_benefit` RPC into `redemptions`. The
  dashboard's "Member Benefits" preview is now DB-driven too. Adds a `category`
  column to `partners`, a `redemption_note` to `benefits`, and a curated 8-partner
  seed. Migration `20260612180000_benefits_phase3.sql`. (Per-level gating lands
  with the admin levels/plans work.)

### Removed
- The **companion card** option on My Card (the "A second card" section, its
  order modal, and the `order_companion_card` RPC). Migration
  `20260612170000_drop_companion_card.sql`.

### Added
- Self-hosting on **Kinetic Cloud** (Enhance) at **membership.gaviso.one**, with
  **auto-deploy on push**: a GitHub Action (`.github/workflows/deploy-kc.yml`)
  SSHes into the box and runs `scripts/redeploy.sh` (pull → build → self-restart).
  Includes `kc:build`, `scripts/deploy.sh`, and a Deploy runbook. (Vercel retired.)
- Member portal redesign **phase 2 — My Card**: report-lost (deactivates the
  card → status `pending` + a history entry), companion-card order, and a real
  **card-history timeline**. Backed by a `card_events` table + `report_card_lost`
  / `order_companion_card` security-definer functions; new signups get a
  "Membership started" event.
- Member portal design refresh (phase 1 of the interactive-portal redesign):
  Gaviso One logo in the hero, official Apple/Google **Add to Wallet** badges,
  a **toast** system, "View your Digital Wallet Profile" → id.gavi.so, and a
  **clickable onboarding** strip wired to real `memberships` flags (adding a
  wallet pass and visiting Benefits now check off, via a `mark_onboarding`
  security-definer function).

### Changed
- Serif typeface changed from **Cormorant Garamond** to **Libre Baskerville**
  across the member portal and admin console (the `--font-serif` token).
- Membership model moved from a binary **founders/member** tier to four named
  **levels** driven by the `plans` table: **Founding Member** ($79), **Starter**
  ($99), **Premium** ($299), **Plus** ($499). The portal now shows the member's
  level everywhere ("You're a Gaviso One {level}", per-level price); My Card drops
  the separate "Cohort" row. New signups default to Founding Member. Migration
  `20260612160000_membership_levels.sql`.
- Admins are now a separate role from members: admin accounts can only access
  the admin UI (they're redirected out of the member portal), and they're
  excluded from the members list and member counts. (The redundant "Admin" link
  in the member nav was removed.)
- The dashboard's "Member Benefits" preview now pulls the three latest items
  from the shared Benefits catalog (single source of truth), and "View all
  benefits" links to the Benefits page.

### Added
- Admin settings (`/admin/settings`): admins can update their username (display
  name), email (with confirmation), and password.
- Admin console (`/admin`), gated to `is_admin` profiles via RLS: **Overview**
  (member/active/pending/partner stats), **Members** (all members with tier,
  status, card, plan, joined), and **Partners** (partners + benefits). Read-only
  for now. Admins also get an "Admin" link in the member portal nav.
- Sign-in method choice on the login page: members can sign in with
  **password** or a **magic link** (passwordless email link via Supabase OTP).
  The `/auth/confirm` route now also handles the magic-link code exchange.
- Edit profile page (`/dashboard/account/edit`): avatar upload, first/last name,
  email (read-only, with lock), phone, website, and mailing address (country as a
  select). Styled with the design bundle's **Treatment A — outlined** form fields
  (hairline borders, ink focus ring, locked-field gray fill, "Optional" tags).
  Linked from the Account section and My Card. Adds profile columns + a public
  `avatars` storage bucket with owner-scoped RLS
  (migration `20260611040000_profile_fields_and_avatars.sql`).
- Member portal pages, with the tab nav wired into real routes and a shared
  hero/nav shell (`MemberShell`): **My Card** (`/dashboard/card`, editorial
  layout — card showcase, wallet/Swft, details, manage, companion card, history),
  **Benefits** (`/dashboard/benefits`, spotlight layout with curated partner
  cards), **Account** (`/dashboard/account`, profile/billing + security), and
  **Support** (`/dashboard/support`). All ported from the Claude Design bundle,
  responsive, auth-gated, and reading the member's real data.
- Supabase membership schema (v1 foundation), provider-agnostic:
  `profiles`, `plans`, `memberships`, `cards`, `partners`, `benefits`,
  `redemptions` with Row Level Security, an admin helper, a `handle_new_user`
  trigger that provisions profile/membership/card on signup, seed reference
  data, and a backfill for existing users. Migration:
  `supabase/migrations/20260611030000_init_membership.sql`.
- Dashboard now reads real membership data from Supabase (with a graceful
  fallback to mock data until the migration is applied).
- Member dashboard at `/dashboard` (auth-gated) implementing the Gaviso One
  "Variation A · Horizontal Tabs" design: dark welcome hero, tab bar, onboarding
  strip, card status with matte-black card mock + wallet passes, member benefits,
  quick links, and account/billing. Brand tokens and fonts (Plus Jakarta Sans,
  Cormorant Garamond) ported from the Claude Design handoff; responsive via real
  media queries. Membership/card data is currently mocked. Login now lands here.
- GitHub Action (`sync release to Quackback changelog`) that posts published
  GitHub Release notes to the n8n webhook, which creates a matching entry on the
  Quackback "Gaviso One Membership" board. Closes the loop: tag → Release → Quackback.
