### Added
- Admin console **Members are now live data** (real profiles/memberships/cards,
  admins excluded) so **Open member portal** opens an actual member's portal.
  Dashboard KPIs/recently-joined reflect real members too.
- Admin **“Open member portal”** (view-as-member preview): from a member's
  drawer, an admin opens that member's portal **read-only** in a new tab. Driven
  by a `gaviso_preview_member` cookie that `requireMember` reads — admins load
  the target member's real data (profile/membership/card/benefits) via the
  is_admin RLS, with a preview banner + Exit. Write actions (edit profile,
  change email/password, support, redeem) are disabled under preview.
- Admin member drawer: **Edit profile** and **Reissue card** actions (sample
  data). Edit updates name/email/company/plan/address; Reissue opens a
  reason picker (lost/damaged/name change, with fee) and queues a new card
  request, flipping the member's card status to Requested.
- Admin **sign out** (rail + My profile) — real `supabase.auth.signOut()` →
  login. And the console now uses the **real signed-in admin's identity**
  (rail avatar, My profile, Admins “(you)”) instead of the sample admin.
- Per-offer **"On dashboard" toggle** (Phase 5b, real data): each partner
  offer in the admin Benefits view (now reading the **live** `benefits` table)
  has a switch controlling whether it appears in the member dashboard's
  "Member Benefits" preview. Admin Benefits is wired to real data — toggle,
  pause/resume, edit offer (incl. levels), and add partner persist to Supabase.
  Column `benefits.show_on_dashboard` (default true); dashboard preview filters
  on it. Migration `20260612250000_benefit_show_on_dashboard.sql`.
- Member benefit **level-gating** (Phase 5a, real data): the member-portal
  Benefits page and dashboard preview now hide partner offers a member's
  level can't access. `benefits.levels text[]` (NULL = all levels) drives it;
  `getBenefits`/`getMember` filter by the member's level. Migration
  `20260612240000_benefit_levels.sql` (Figma & Ace Hotel set to Premium/Plus).

### Changed
- **Business profiles** cards (Account) redesigned to the structured layout: a
  header row (logo, name + Primary pill, role · industry, actions) over a tinted
  detail grid with labeled **Contact / Address / Social** columns and
  platform-labeled social chips (e.g. INSTAGRAM @acmeco).
- **Sign-in page redesigned** to the design bundle's split layout: a dark left
  welcome panel (serif “Welcome back.”, lede, member-card mock) and a right
  sign-in panel. Keeps real auth — email **password** or **magic link**
  (show/hide password, remember me, forgot password), admins → /admin and
  members → /dashboard. New `app/auth/login/{layout,login.css}`.

### Fixed
- The member portal now shows each member's **Member ID** in the Account
  Membership card, and `getMember` is scoped with `.eq(id/user_id)` instead of
  relying on RLS returning a single row — admins (who can see every row) no
  longer trip a `maybeSingle` error that silently fell back to mock member data.

### Added
- Admin console **notifications**: the topbar bell opens a popover (unread
  dots, mark-all-read, relative times) where each item jumps to the relevant
  section; the bell nub reflects the unread count. Closes on click-away/Esc.
- Admin console **round 4 — Admins & roles, Settings, Profile** (sample data):
  **Admins & roles** table (Owner/Ops/Support pills, last-active, two-factor),
  admin drawer (change role/email, activity), invite-admin modal with role
  cards. **Settings**: general, card program, a Roles & permissions matrix,
  notification toggles, data exports. **My profile**: editable profile,
  access summary, security, active sessions. The console is fully sample-data;
  real Supabase wiring is the final round.
- Admin console **round 3 — Invoices**: the Billing view now lists member
  invoices (number, member, plan, amount, period, due, Paid/Due/Overdue status),
  rows open the member drawer. (Plans shipped in round 2.)
- Admin console **round 2 — Benefits + Plans** (sample data): **Benefits** view
  with partner-offer cards (status Live/Paused/Draft, redemption method + code,
  **per-level availability**, redemptions), a partner drawer (pause/resume,
  edit offer), add-partner and edit-offer modals with a **level picker**, and
  **GDM products** (toggle live, SSO / sign-in link, add/edit/remove). **Plans**
  under Billing: the four membership levels with price, open/closed status, and
  member counts (add/edit level modal). Shared admin primitives extracted to
  `components/admin/shared.tsx`.
- New **Admin console** (design-bundle rebuild, UI-first on sample data) —
  sidebar-rail SPA replacing the old `/admin` overview. This round ships the
  shell + **Dashboard** (KPIs, card-pipeline funnel, recently-joined, activity),
  **Members** (filters, search, detail drawer, suspend/reinstate), and **Cards**
  (queue by status, drawer to set status / paste tracking / cancel). Benefits &
  plans, Billing, and Admins & roles are stubbed for the next rounds. Data is the
  design's sample set (`lib/admin/admin-data.ts`); real Supabase wiring comes
  per-section after.
