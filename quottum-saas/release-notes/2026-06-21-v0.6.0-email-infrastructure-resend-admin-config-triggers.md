The platform now sends real email. Resend integration, platform-tier config in \`/admin/email\`, inline triggers on quote status transitions. Per-workspace template overrides + cron-based quote-expiry reminders queued for v0.6.1.

### Schema (migration 016)

\`\`\`
admin_email_settings        (singleton: provider, api_key, default_from)
workspace_email_settings    (per-workspace from override)
email_templates             (workspace_id nullable — workspace row wins,
                             NULL row = platform default)
email_log                   (per-send record: recipient, template, status,
                             error, message_id)
\`\`\`

Five platform-default templates seeded: \`quote.sent\` · \`quote.accepted\` · \`quote.declined\` · \`renewal.reminder\` · \`quote.expiring\`. Each uses \`{{variable}}\` substitution — no expressions, no escaping, just literal replacement.

### sendEmail helper

\`\`\`ts
import { sendEmail } from '@/lib/email'

await sendEmail({
  workspaceId,
  templateKey: 'quote.accepted',
  to: 'client@example.com',
  vars: { workspace_name: 'Foo Studio', quote_number: 'Q-0042', … },
})
\`\`\`

Resolves provider + sender + template, renders, sends via Resend, writes the log row. Never throws — returns \`{ok, messageId, error}\`. Failures land in the log so an operator can investigate without the user action breaking.

### /admin/email

New page in the operator sidebar. Provider dropdown (Resend now; Postmark / SES placeholders), sender domain, default from-email/name, **write-only** API-key field (masked on read as \`•••• {last 4}\`), DKIM-verified chip, **Send test** button (prompts for recipient), and the most-recent 20 sends across all workspaces.

### Inline triggers

The three quote status transitions now fire email immediately:

- **Dashboard → Publish → \`status='sent'\`** → \`quote.sent\` to the client
- **Public quote → Accept** → \`quote.accepted\` (with contract link)
- **Public quote → Decline** → \`quote.declined\`

All routed through the helper, so a misconfigured provider just produces a \`failed\` row in \`email_log\` — the user-facing action still succeeds.

### What you need to do to actually send

1. Sign up for Resend, verify your sending domain
2. Open https://app.quottum.io/admin/email
3. Paste your Resend API key, set the default from-email + sender domain, **Save**
4. Hit **Send test** — should land in your inbox within seconds
5. Now publish a draft quote → the client gets the \`quote.sent\` email automatically
