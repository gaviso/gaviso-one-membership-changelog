The customer-facing contract surface plus the coupon model goes from \"two checkboxes that don't enforce anything\" to actually-enforced.

### Public contract page — /c/[token]

A read-only public view for any contract with a non-null \`client_token\`. Mirrors the \`/q/[token]\` quote layout: workspace branding, headline pricing (net of any recurring discount), term card, notes, workspace-wide terms, contact + download-PDF footer. Linked from the client portal (\`/p/[token]\`) so a customer can drill from their portal into any of their contracts.

### Contract PDF — /contracts/[id]/pdf

Print-friendly server-rendered page, same shape as the quote PDF. Workspace branding header, **Between / And** party block, subject, full terms grid (billing · term · fee · discount line when set · MRR/ARR/TCV · dates · auto-renew · status), notes + terms blocks via the same \`<RichTextRender>\` used elsewhere, signature lines for both parties, footer. Download-PDF button in the contract detail topbar.

### Coupon enforcement

Coupons were toggleable (single_use / recurring) since v0.3.0 but neither flag actually enforced anything. Now they do:

- **`single_use`** — backed by a real \`coupon_redemptions\` ledger. One row per accepted quote. Unique \`(coupon_id, quote_id)\` constraint makes re-accepts idempotent. Re-saving an already-redeemed quote uses \`skipQuoteId\` so it doesn't trip its own gate.
- **Client scope** — new \`coupons.client_id\` (nullable FK). Null = workspace-wide. Set = only valid on quotes for that client.
- **Expiry + active** — validated server-side; the public quote-accept route refuses to redeem a stale or inactive coupon.
- **Recurring** — when the coupon is recurring and the quote is accepted, the auto-spawned contract inherits \`coupon_id\` / \`coupon_code\` / \`coupon_discount\`. The renewal flow (\`POST /api/contracts/[id]/renew\`) carries it onto each renewal so the discount keeps re-applying. Non-recurring coupons drop off at renewal so price reverts to list.

All four checks happen in **\`validateAndPriceCoupon()\`** (\`src/lib/coupons.ts\`), called from both \`POST\` and \`PATCH\` of the quote API.

### Auto-contract math (bonus fix)

Quote \`accept\` used to hard-code \`billing_cycle='monthly'\` and \`term=12\` regardless of the quote's actual billing period. The spawned contract now reads \`billing_period\` and \`term_months\` from the quote and computes MRR/ARR/TCV correctly per cycle (one-time / monthly / quarterly / annual). Currency is also threaded through (was silently USD).

### Schema (migration 015)

\`\`\`
coupon_redemptions (id, coupon_id, quote_id, redeemed_by, redeemed_at,
                    unique(coupon_id, quote_id))
coupons.client_id           uuid references clients
contracts.coupon_id         uuid references coupons
contracts.coupon_code       text
contracts.coupon_discount   numeric default 0
\`\`\`
