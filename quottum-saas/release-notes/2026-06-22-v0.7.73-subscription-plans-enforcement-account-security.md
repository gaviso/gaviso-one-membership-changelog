The biggest release yet — subscription plans, runtime plan enforcement, and a full account-security layer, plus a sweep of on-brand polish.

## Subscription plans
- **Plan catalog** in the operator console with a **feature matrix** — define which features each plan includes, grouped by collapsible category with descriptions on hover.
- **Per-plan limits** (clients, seats, quotes/month, workspaces) managed right in the matrix.
- **Per-interval pricing** — monthly / 3-month / 6-month / annual prices per plan.
- Assign a plan to a workspace with a **billing cadence**; the renewal date auto-fills from the cadence.

## Plan enforcement (new)
- Plans are now **enforced at runtime** — feature gates (contracts, exports/imports, renewals, multiple workspaces) and count limits (clients, seats, quotes/month).
- **Non-destructive**: existing data is always kept; only new actions past a limit are blocked.
- **Downgrade preview** for operators (what locks / goes over-limit before assigning a lower plan) and an **over-plan banner** for affected workspaces. Upgrade locks appear on gated buttons.

## Account & security
- **Forgot-password flow** — request a reset link and set a new password.
- **Magic-link sign-in for customers** (passwordless email link), alongside password.
- **User activity log** — sign in/out, sign-ups, and password resets with IP + approximate location, both console-wide and per workspace (covers magic-link logins too).
- **Operator profile** — editable name/email/avatar, appearance (system/light/dark), and per-account two-factor auth.

## Polish
- Sign-in screens and shared buttons are now **on-brand green** (no stray indigo); consistent, readable settings buttons & nav.
- Workspace members show real **names/emails** instead of IDs; colored initial avatars for clients & workspaces.
- Route-level **loading skeletons**, rebuilt favicon, and a release timeline + GitHub changelog in the console.
