First production deploy of Quottum SaaS to **[app.quottum.io](https://app.quottum.io)** on Kinetic Cloud (Enhance), with push-to-deploy from GitHub.

### Added

- **Kinetic Cloud push-to-deploy.** [`deploy-kc.yml`](https://github.com/gaviso/quottum-saas/blob/main/.github/workflows/deploy-kc.yml) GitHub Action SSHes into the Enhance Node host on every push to `main` and runs [`scripts/redeploy.sh`](https://github.com/gaviso/quottum-saas/blob/main/scripts/redeploy.sh) — `git pull` → `npm ci && next build` → signal the supervised process to restart.
- **`kc:build` script** in `package.json` and a self-contained `redeploy.sh` that works the same way over interactive SSH or from CI.
- **[DEPLOYMENT.md](https://github.com/gaviso/quottum-saas/blob/main/DEPLOYMENT.md)** fully rewritten for the production path — DNS, Enhance Node.js website setup, SSH deploy key, server bootstrap, the `.env.production` build-time gotcha, GitHub secrets, env-var matrix, cron schedule, smoke test.
- Internal-wiki page documenting this pattern for future apps.

### Fixed

- **Production build failure** — `StatusBadge` still imported `STATUS_COLORS` after it was replaced by `STATUS_TONE` in v0.2.0; rewritten to render via the `wf-chip` Chip primitive so it matches the rest of the dark design system.
