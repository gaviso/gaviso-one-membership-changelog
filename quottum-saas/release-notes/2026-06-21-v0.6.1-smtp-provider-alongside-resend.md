Operators can now configure plain SMTP as a sibling to the Resend integration shipped in v0.6.0. Useful for Gmail / Microsoft 365 / Mailgun-via-SMTP / self-hosted / any RFC-5321 server.

### What you can do now

Open https://app.quottum.io/admin/email → **Provider** dropdown now has two options:

- **Resend (API)** — the v0.6.0 default. Paste your \`re_...\` API key.
- **SMTP (any RFC-5321 server)** — host / port / encryption / username / password.

The form swaps fields based on which one you pick. Encryption picker presents the two sane options: **TLS on 465** (implicit TLS) or **STARTTLS on 587** (upgraded TLS).

### Schema (migration 017)

\`\`\`
admin_email_settings: + smtp_host, smtp_port (default 587),
                      + smtp_secure (default true),
                      + smtp_user, smtp_pass
\`\`\`

Both credential sets live in the same singleton row; the helper picks based on \`provider\`.

### sendEmail() now dispatches on provider

- \`'resend'\` branch → Resend SDK (unchanged from v0.6.0)
- \`'smtp'\` branch → nodemailer's \`createTransport\` with host/port/secure/auth

Both write to \`email_log\` the same way; failures land as \`failed\` rows so the operator can investigate.

### Examples

| Provider | Host | Port | Encryption | Username |
|---|---|---|---|---|
| Gmail / Workspace | smtp.gmail.com | 587 | STARTTLS | your@workspace.com (use an app password) |
| Microsoft 365 | smtp.office365.com | 587 | STARTTLS | your@org.onmicrosoft.com |
| Mailgun | smtp.mailgun.org | 587 | STARTTLS | postmaster@mg.your-domain.com |
| Postmark | smtp.postmarkapp.com | 587 | STARTTLS | (server API token both fields) |
| Self-hosted | smtp.your-mailserver.com | 465 | TLS | your-mailbox |

Deps: \`nodemailer\` + \`@types/nodemailer\`
