## "New version available" banner

Every open tab now learns about a new deploy within 5 minutes and shows a small top-of-page banner:

> 🔄 **A new version is available** (v0.7.X, you're on v0.7.Y) · **Reload** · ✕

### How it works
- `next.config.ts` bakes `pkg.version` into `NEXT_PUBLIC_APP_VERSION` at build time → inlined into both bundles. Client bundle reads the constant at module init (so the tab knows what version it was built with). New `/api/version` endpoint returns the *currently deployed* version at request time.
- After a deploy: the server bundle returns the new version; old tabs still hold their old client constant → mismatch → banner.

### Behavior
- **Polls every 5 min** while the tab is visible. Pauses on hide (Page Visibility API), refreshes + resumes on focus — idle tabs don't hammer the endpoint.
- **Dismiss is per-session AND per-version**: dismissing v0.7.X doesn't suppress a later v0.7.Y. Resets on tab close (next session, the banner reappears).
- **Reload click** → `window.location.reload()`. Hashed chunk URLs handle cache invalidation automatically. Anyone mid-edit who wants to defer can dismiss instead.
- **Dev builds** (`NEXT_PUBLIC_APP_VERSION=local`) suppress the banner so every save doesn't flag "new version".
- Mounted on both customer dashboard + operator `/admin` layouts.

### One bug caught + fixed pre-release by adversarial review
- The first version had a sessionStorage flash race: the dismiss flag was checked when `serverVersion` was still null (so the comparison always failed), so a dismissed banner would briefly re-appear on tab reopen before the second `useEffect` fixed it. Refactored so `setServerVersion` and `setDismissed` fire in the same tick — no flash.
