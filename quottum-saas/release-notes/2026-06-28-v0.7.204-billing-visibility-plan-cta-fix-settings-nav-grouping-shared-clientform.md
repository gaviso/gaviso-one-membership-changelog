Four UX/polish changes across billing, settings, and the client forms.

### Billing — feature visibility

- **"What you have" is now a 3-column responsive grid** (1 col mobile → 2 → 3 on desktop), keeping the "Granted by support" chips and expiry suffixes.
- **New "Unlock more with an upgrade" section** — lists the features the current plan *doesn't* include, grouped by the cheapest purchasable plan that unlocks each, so the user sees exactly which plan to move to (e.g. for a Free workspace: "Pro unlocks 18 features", "Business unlocks 4"). Excludes operator-blocked features (those stay under "Temporarily unavailable") and comp/beta plans; hidden on the top plan and for non-elevated members.

### Billing — current-plan CTA fix

The plan picker showed "Start 14-day trial" on the card for the workspace's own current *active* plan — even though the "CURRENT PLAN" badge rendered correctly. The badge and the CTA logic disagreed: the CTA had no branch for "this is the current active plan" and fell through to the trial CTA (keyed purely on `trial_days > 0`). Added a `currentPlanStatus` prop and a short-circuit so the current active plan shows a disabled "Current plan" button instead. Trialing / pre-checkout flows and other plans' trial CTAs are unchanged.

### Settings — sub-nav grouping

Settings sidebar items now cluster tightly **within** each section (General / Appearance / Branding under Workspace; Categories / Service Groups / Line items under Catalog; …) while the section dividers keep the groups clearly separated. Achieved by tightening the per-item vertical padding (`.quottum-set-nav .item`, 10px → 5px) so the contrast — ~2px between sibling items vs ~26px + a divider between groups — does the visual grouping.

### Clients — shared ClientForm (new page matches edit page)

The new-client page was still pre-design-system (white/gray cards, off-brand indigo focus rings, the old `@/components/ui` inputs) while the edit page had been modernized. Rather than restyle in place, extracted a shared `ClientForm` component used by **both** pages, so they can't drift apart again:
- New `src/components/clients/ClientForm.tsx` renders the sectioned field cards.
- The edit page was refactored onto it with zero behavior change (PATCH, "Save & start quote", the Classification sidebar, loading state all preserved).
- The new page was rebuilt on the design system with a modern header (back pill, Avatar, "New client" eyebrow, serif title), single column (no stats sidebar — a brand-new client has none), and "Save & start quote" wired to `POST /api/clients` → redirect to the new client or straight to quoting them.

### No migration

Pure code, type-checked clean on the composed tree. Built across parallel background agents (sidebar, ClientForm) with the billing work and a current-plan CTA fix, then consolidated into one release.
