Bug-sweep release driven by the post-v0.4.8 code review. Three security issues, several user-facing breakage fixes, and a vocab cleanup.

### Critical (security)

- **SSRF guard on /api/cron/test-event** — the endpoint was accepting an arbitrary URL in the request body and POSTing to it. Any logged-in user could use it as a relay against internal hosts. URL is now read server-side from the workspace's configured \`webhook_url\` / \`slack_webhook_url\`.
- **Cross-workspace data wipe on PATCH /api/quotes/[id]** — the route was running \`DELETE quote_items WHERE quote_id = ?\` filtered only by quote id, before the workspace-scoped quote update. A logged-in user could nuke another workspace's line items by passing its id. Workspace ownership is now verified first.
- **Tightened proxy public-allow rule** for \`/api/invitations/\` — was \`startsWith('/api/invitations/')\`, would silently expose any future handler in that tree. Now an exact per-token regex.

### High (user-facing breakage)

- **Public quote page (`/q/[token]`)** 500'd on unknown tokens because of \`.single()\`. Switched to \`.maybeSingle()\` then \`notFound()\`. Same page now threads the workspace currency through every \`formatCurrency\` call.
- **PDF rendered TipTap HTML as raw markup** — descriptions and notes printed as \`<p>foo</p>\`. Now route through the same \`<RichTextRender>\` used elsewhere. Public quote page too.
- **ContractForm was missing the \`semiannual\` billing option** and had no currency picker — every contract silently stored the DB default. Both added. \`POST /api/contracts\` now persists \`currency\` through.
- **ContractForm now does \`router.refresh()\`** before push so the destination contract page isn't served from stale RSC cache.

### Cleanup

- Dropped orphan \`/dashboard/settings/company\` page (replaced by Branding ages ago, never linked).
- Swept every API route's \"No org\" error string → \"No workspace\".
- Hardened impersonation cookies with \`secure: NODE_ENV === 'production'\` so they only ride HTTPS in prod.

### Not in this release (queued)

The review surfaced a wider currency-threading gap on the dashboard/contracts list and the admin metrics page (rows from multi-currency workspaces silently render as USD). Tracked as a follow-up — needs per-row currency carried through.

### Up next

**v0.5.0 — Public contract docs + coupon enforcement** (deferred while we cleared the sweep). The currency-threading task lands as part of that pass.
