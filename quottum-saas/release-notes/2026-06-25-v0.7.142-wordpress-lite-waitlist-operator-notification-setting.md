## WordPress Lite waitlist
First-party signup form for the free Quottum WordPress plugin. Replaces the previous Waitlister.me embed with a form that captures **5 questionnaire answers** alongside name + email — so launch ramp can prioritize from data we own.

**Public form** lives at [quottum.io/wordpress/waitlist](https://quottum.io/wordpress/waitlist) (marketing site). It POSTs cross-origin to `app.quottum.io/api/wp-lite-waitlist` — CORS pinned, honeypot bot screen, server-side validation, duplicate-email idempotent.

**Operator console** at **/admin/wp-lite-waitlist**:
- Sortable list of all signups, urgency chips (red for "I need it now")
- CSV export for the full dataset
- New sidebar entry under Beta testers

**Notification email** is now editable from the operator console (Resend card on `/admin/integrations` → "Operator notification email"). Falls back to the `OPERATOR_NOTIFICATION_EMAIL` env var when blank, so the v0.7.141 setup keeps working without operator action.

## Fix
- Patched the proxy auth gate so `/api/wp-lite-waitlist` is reachable cross-origin without a session (the route's own CORS + honeypot + validation are the gate).
