Everything billing-related now happens in-app — the Stripe Customer Portal button is gone.

### Change plan (in-app)
- Wired the previously-unused `/api/billing/change-plan` to a new modal: pick plan + cadence, see the proration consequence, confirm.
- **New proration policy — both directions immediate, both prorate** (`proration_behavior: 'always_invoice'`, no billing-cycle reset):
  - **Upgrade** → prorated difference charged now; unused old-plan time credited against it; new features immediate.
  - **Downgrade** → cheaper plan effective immediately; unused higher-plan time nets **account credit** to `Customer.balance` → `workspaces.credit_balance_cents`, applied to future invoices.

### Cancel → cancel at period end + Resume
- Self-serve **Cancel** now schedules `cancel_at_period_end` — **no refund, no credit**; the workspace keeps full access until `current_period_end`, then doesn't renew.
- New **Resume** action un-schedules it while the sub is still active.
- The **operator** cancel (admin) is unchanged — it still prorates + issues credit.

### Stripe portal removed
- Deleted the "Manage billing in Stripe" button + `/api/billing/portal`. Cancel, resume, payment method, invoices, and plan change are all in-app. Tax-ID / billing-email management (portal-only) is dropped.

Built across 3 background agents (Change-plan/portal, cancel-at-period-end/Resume, + the inline proration edit), reviewed, composed type-check + lint clean. Policy recorded in AGENTS.md + memory.
