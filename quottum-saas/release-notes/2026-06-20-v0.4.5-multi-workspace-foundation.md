The load-bearing rename: every user now has a **workspace** (no more \"organization\" anywhere), and a single user can belong to multiple workspaces. Foundation for the admin surface (v0.4.5-b/c) and everything downstream.

### Database

- Migration 008: rename **organizations → workspaces**, **organization_members → workspace_members**, **org_settings → workspace_settings**. Every \`org_id\` column renamed to \`workspace_id\` (12 tables). Helper functions, FKs, indexes, and all 20 RLS policies recreated under the new names. **Single transaction — no half-renamed state.**
- New workspace lifecycle columns on \`workspaces\`: \`status\` (\`active\` / \`trial\` / \`suspended\` / \`archived\`), \`plan\` (\`free\` by default), \`limits\` (jsonb), \`trial_ends_at\`. Wired now so future billing/lifecycle work doesn't need another migration.
- Migration 009: \`workspace_invitations\` table with token + 14-day expiry.

### Signup

- Single-step: first name + email + password.
- Auto-provisions a \"{First Name}'s workspace\" with a random-suffix slug, drops the user straight into the dashboard.
- Snapshots full name into \`auth.users.user_metadata\` so activity-timeline attribution falls back to it.

### Workspace switcher

- Dropdown above the sidebar nav listing every workspace the user belongs to with a checkmark on the current one and a **\"Create workspace\"** inline form.
- Cookie-driven: \`quottum_workspace\` cookie wins when set; otherwise oldest membership (so single-workspace users see no behaviour change).
- Switch endpoint verifies membership before writing the cookie — a malicious client can't enumerate workspaces.

### Invite flow

- Settings → Permissions: members list + invite form + pending-invitations table with **Copy link** and **Revoke** actions.
- Public \`/invite/[token]\` page shows workspace + role, prompts sign-in/sign-up if needed.
- Accept endpoint enforces email-match — a leaked token can't be redeemed by an unrelated account.
- **Email delivery deferred to v0.6.0** — for now copy the invite link from the pending list and share it manually.

### Suspension gate

- New \`/suspended\` page that renders **only** when the user's resolved workspace is actually in a blocked state (no spoofable query params).
- Dashboard layout redirects to it when \`workspace.status\` isn't \`active\` or \`trial\`.

### Code sweep

- 38 source files updated mechanically: \`getUserOrg → getUserWorkspace\`, \`member.org → member.workspace\`, \`.eq('org_id', …) → .eq('workspace_id', …)\`, RPC \`p_org_id → p_workspace_id\`. Types renamed: \`Organization → Workspace\`.
