Big batch covering client-facing UX, operator workflow cleanup, settings reorg, and several correctness/security fixes — including a critical PATCH allowlist that closes a hole the adversarial review caught on the contract toggles.

## 🎯 Client portal & public quote view

- **Redesigned client portal** ([v0.7.157](https://github.com/gaviso/quottum-saas/commit/b6eece7)) — "Welcome back, {name}" greeting, quote cards with status pills + total + valid-until + line item count, "View quote" CTA. Contracts in a clean table layout. Warm "X awaiting your approval" banner when there's actionable work.
- **Clickwrap terms checkbox** ([v0.7.153](https://github.com/gaviso/quottum-saas/commit/01681d4)) — terms now render ABOVE the Your-response card with a required checkbox gating the Accept button. Decline and Request revision stay un-gated (those don't bind the client).
- **Attachment preview modal** ([v0.7.154](https://github.com/gaviso/quottum-saas/commit/b6a0e99)) — clicking image/PDF attachments on public quote/contract views opens an in-page preview instead of a new tab. Other types still download.

## 📝 Operator quote management

- **Removed operator Accept/Decline** ([v0.7.156](https://github.com/gaviso/quottum-saas/commit/00bcbec)) — those are the client's decision, taken via /q/<token>. Operator UI shows Re-send / Revise only.
- **View contract button on accepted quotes** ([v0.7.163](https://github.com/gaviso/quottum-saas/commit/6b1884b)) — quotes list preview pane now links directly to the auto-spawned contract for accepted quotes. Fixed in [v0.7.164](https://github.com/gaviso/quottum-saas/commit/f52e3b6) to handle renewal chains correctly (the link now resolves to the most recent contract).
- **In-app preview links use relative paths** ([v0.7.155](https://github.com/gaviso/quottum-saas/commit/6cc4aa5)) — "Client view" and "see what it looks like" buttons no longer depend on NEXT_PUBLIC_APP_URL, which had broken in dev when ports didn't match.
- **Contracts table fits without scroll** ([v0.7.156](https://github.com/gaviso/quottum-saas/commit/00bcbec)) — merged End-date + Days-left into a single "Ends" column, replaced dedicated Auto-renew column with an inline chip, smaller avatar, compact padding.

## 📁 Contracts: line items, edit gating, renewal at catalog price

- **Service group + description on contract line items** ([v0.7.164](https://github.com/gaviso/quottum-saas/commit/f52e3b6)) — contract detail page now uses the same renderer as the operator quote view: ColorChips for both Category and Service group, expandable description drawer. Both columns previously showed bare dashes.
- **Toggles actually work** ([v0.7.165](https://github.com/gaviso/quottum-saas/commit/6e9c756)) — Auto-renew and Renew-at-catalog-price toggles in the Term & billing card were display-only widgets. Both are now real interactive switches with optimistic UI + revert-on-error.
- **Renew at catalog price built out end-to-end** ([v0.7.165](https://github.com/gaviso/quottum-saas/commit/6e9c756)) — when the flag is on, opening "Schedule renewal" recalculates the new amount against current library prices. Form shows a green notice strip with the suggested amount, items the library re-priced, and warnings for items no longer in the library. Operator can still override.
- **Quick edit only on active contracts** ([v0.7.167](https://github.com/gaviso/quottum-saas/commit/177d8e2)) — Option B from the operator's choice. Active/signed contracts can only edit title, notes, and renewal toggles. Financial terms (amount, billing, dates, client) are locked — substantive changes go through Renew or Terminate. Yellow notice explains the gating.
- **🔒 PATCH /api/contracts/[id] allowlist** ([v0.7.167](https://github.com/gaviso/quottum-saas/commit/177d8e2)) — **security fix**. The adversarial review caught that the PATCH endpoint was accepting ANY field. Two-tier allowlist (ALWAYS_SAFE + DRAFT_ONLY) now enforces what's editable based on contract status. Closes vectors for revenue zeroing, audit bypass, client reassignment, token swap, and cross-workspace moves.
- **calcRevenue deduplicated** ([v0.7.167](https://github.com/gaviso/quottum-saas/commit/177d8e2)) — three copies across the contract API routes all silently missed the semiannual case. Now a single helper at lib/contract-revenue.ts with all 6 billing cycles correct.
- **Attachments moved to main pane + wider Service group column** ([v0.7.166](https://github.com/gaviso/quottum-saas/commit/e87f195)) — drop zone gets room to breathe; long chip names like "Retainer support" fit on one line.
