A large batch built across ~14 agents on a feature branch, reviewed + committed per task.

## Billing UX
- **Confirm-then-pay** on both surfaces. Change-plan: select → preview (amount due − credits) → confirm/pay; charges confirm a PaymentIntent via Stripe Elements (saved card selectable via CustomerSession), declines recoverable; credit/deferred are confirm-only. Subscribe: clicking a plan no longer creates a subscription — payment is created only on explicit "Continue."
- **Yearly-downgrade deferral**: on a yearly plan any downgrade defers to renewal via a Stripe Subscription Schedule (no double-charge); upgrades + monthly immediate. Preview + apply share one classifier.
- **Operator-initiated, owner-completed** plan change: the operator stages a proposal on a live-Stripe workspace (no direct Stripe mutation); the owner gets a one-shot dashboard popup and completes it through the confirm-then-pay flow.
- **Payment method section** (card on file + Update). Removed the redundant "View plans" link.

## Bug-fix sweep (full user + operator audit)
- **CRITICAL:** public quote accept/decline/revise silently no-op'd (anon client vs RLS) — now persists + creates the contract.
- **HIGH:** client_token fixation via mass-assignment; branding writes not role-gated; the renewals cron ran as the anon client (reminders were a no-op); comp/set_plan accepted unvalidated plan keys.
- **MEDIUM/LOW:** revise-status, expired-accept, cross-workspace client PII, CSV formula-injection, invites/catalog authz, TOS spoof, CRON_SECRET fail-open (all 6 crons), Stripe price archiving on plan deactivate, dead quarter/half-year pricing, beta grant/reject hardening, broken public links.
- Operator admin auth verified clean (all 47 admin routes).
