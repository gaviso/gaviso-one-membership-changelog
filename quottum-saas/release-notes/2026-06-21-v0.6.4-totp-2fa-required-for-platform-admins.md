Sign-in to the operator console now requires a second factor. Operator accounts hold service-role keys, can impersonate any workspace, and read everyone's data — one phished password should not be game over.

> **Heads-up:** TOTP MFA must be enabled at the **Supabase project level** (Authentication → Multi-Factor Authentication → TOTP). Default in newer projects; existing projects may need a flip. The first time you visit \`/admin\` after deploying you'll be sent to \`/operator/enroll\` to set up an authenticator app.

### Schema (migration 018)

\`\`\`sql
platform_admins.mfa_required boolean not null default true
\`\`\`

Per-admin opt-out via SQL if needed (incident response, onboarding window). Backfilled to \`true\` for existing admins.

### Flow

1. Sign in at \`/admin/login\` (password **or** magic link from v0.6.3) → session is **AAL1**
2. \`/admin\` layout checks:
   - \`mfa_required\` and no verified factor → \`/operator/enroll\`
   - Has verified factor and \`currentLevel=aal1\` → \`/operator/challenge\`
3. After verify, session is **AAL2** → \`/admin\` renders

### \`/operator/*\` pages

Live at the top level (not under \`/admin\`) so the AAL2 gate doesn't infinite-loop them. Their own layout still gates on \`platform_admin\` — non-admins never see them.

- **\`/operator/enroll\`** — runs \`mfa.enroll\` → shows QR code + secret + 6-digit input. Strips any unverified orphan factors on mount so re-enroll attempts don't pile up. Works with 1Password, Authy, Google Authenticator, Raycast, etc.
- **\`/operator/challenge\`** — \`listFactors\` → grabs the verified TOTP factor, runs \`mfa.challenge\` + \`mfa.verify\` on a 6-digit code. Honors \`?next=\` with a same-origin guard.
- **Escape hatch** — "Sign out instead" link below both pages. If the operator lost their phone or signed in with the wrong account, they need to get out without verifying.

### \`/admin/security\`

New nav entry in the operator sidebar.

- Enrollment status chip (✓ Enabled / ⚠ Not enrolled)
- List of verified factors with friendly name + enrolled date
- Sign-in methods summary (password, magic link)
- **Re-enroll** link (rotates the secret — replaces the current factor)
- **Disable 2FA** button — gated behind a typed \`DISABLE\` confirm AND only renders when \`mfa_required\` is \`false\` (otherwise the operator would lock themselves out next page load)

### Why \`/operator/*\` (not \`/admin/*\`)

The enroll and challenge pages MUST be reachable at AAL1 — that's the whole point. Putting them under \`/admin\` means the layout's AAL2 gate would redirect them right back to themselves. Same route-group pattern we used for \`/admin/login\` in v0.5.x.
