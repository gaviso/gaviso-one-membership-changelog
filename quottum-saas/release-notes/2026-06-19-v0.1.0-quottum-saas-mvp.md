First public cut of Quottum SaaS — a multi-tenant quoting and contracts platform built on Next.js 16, Supabase, and Tailwind v4.

### Added

- **Auth & orgs** — login, signup, and org-creation flows with Supabase SSR. Every record is scoped to `org_id` with RLS.
- **Clients module** — list, create, and detail pages with full CRUD.
- **Quotes module** — builder with line items, categories, coupons, draft/sent/accepted/declined states, public token-protected client view, and accept/decline/revise actions.
- **Contracts module** — list, detail, and edit pages with MRR/ARR/TCV revenue calculations across monthly, quarterly, annual, and one-time billing cycles. Auto-renew, status transitions, and an event timeline.
- **Settings** — company profile, line-item library, categories, coupons, terms & conditions, renewal-reminder milestones, and webhook/Slack integrations.
- **Renewal engine** — `/api/cron/renewals` sweeps active contracts daily, fires per-milestone webhooks and Slack alerts, with idempotency guards via `renewal_log`.
- **Dashboard** — MRR/ARR/active-contract KPIs, recent quotes, and upcoming renewals (≤90 days) with severity coloring.
- **Quote → Contract** — accepting a quote on the public link auto-creates a contract using the org's default term-months setting.

### Infrastructure

- Supabase schema split across two migrations (`001_init.sql`, `002_contracts.sql`).
- Numbered quote and contract IDs via Postgres functions (`next_quote_number`, `next_contract_number`).
- Webhook + Slack delivery logging in `event_log`.
- Changelog pipeline: GitHub Releases → n8n → Coolify auto-deploy at https://changelog.quottum.io.
