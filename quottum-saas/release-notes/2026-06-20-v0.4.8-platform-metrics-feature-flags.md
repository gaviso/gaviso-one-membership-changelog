Final piece of the v0.4.5 admin trilogy: real platform metrics dashboard + per-workspace feature flag matrix. The operator console now has both visibility (metrics) and control (flags) on top of the actions already shipped in v0.4.7.

### /admin/metrics

A real KPI dashboard, not just an overview snapshot:

- **Top strip**: Workspaces · Platform MRR · Avg MRR/workspace · MAU
- **Two 30-day sparklines** (inline SVG, no chart lib): Signups · New workspaces, each with 7-day rolling totals
- **Quote funnel** (last 90 days): Created → Published → Sent → Accepted with horizontal progress bars + win rate
- **Workspace lifecycle**: active / trial / suspended / archived with 30-day churn percentage
- **Active-contract economics**: TCV, ARR, MRR, avg contract size
- **Top 8 workspaces by MRR** — deep-linked to admin detail

### /admin/flags — workspace × flag matrix

Replaces the v0.4.5-b placeholder. Five starter flags in the catalogue:

- ` + "`beta_admin_panel`" + ` — show the operator surface to a workspace owner
- ` + "`raised_quote_limit`" + ` — lift the free-plan quote ceiling
- ` + "`client_portal`" + ` (default ON) — public /p/[token] portal
- ` + "`webhook_deliveries`" + ` (default ON) — outbound webhooks on quote/renewal events
- ` + "`debug_logging`" + ` — verbose server logs scoped to the workspace

Each row is a workspace, each column a flag. Toggling creates an override (accent ring); the small ↺ button resets to default. Every change logs to admin_audit.

### Helpers (src/lib/feature-flags.ts)

- ` + "`isFeatureEnabled(workspaceId, key)`" + ` — service-role read, falls back to catalogue default if no override row exists
- ` + "`getFlagValue(workspaceId, key)`" + ` — for raised-limit-style jsonb payloads
- ` + "`listFlagsForWorkspace(workspaceId)`" + ` — bulk read
