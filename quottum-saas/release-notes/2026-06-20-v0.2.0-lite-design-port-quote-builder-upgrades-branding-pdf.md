A heavy UI + feature pass that brings Quottum SaaS in line with the Quottum WordPress Lite plugin. Same data philosophy, same dark Quottum design system, plus the quote-builder polish, branding, and PDF output that were missing.

### Design system

- Full **dark Quottum theme** (green accent `#3ecf8e`, Supabase-style) ported in — every primitive available: cards, pill buttons, status chips, KPI stats, tables, segmented controls, switches, date pickers, drawers, timelines, lifecycle strips, settings nav, modal, markdown editor, mobile responsive shell.
- React wrappers under `src/components/wf/*` plus a new `Sidebar` + `Topbar` matching the Lite shell exactly.
- **Plus Jakarta Sans** + **Source Code Pro** via `next/font`.
- **Quottum sprout mark** in the sidebar and as the browser favicon.
- Tailwind palette **remapped at the `@theme` layer** so legacy classes (`bg-white`, `text-gray-*`, `bg-indigo-*`) resolve to the dark palette without touching every old file.

### Dashboard

- Rebuilt as the Lite \"command center\" — KPI strip (TCV / MRR / ARR / Active contracts), contract register table with status chips (At risk / Renews soon / Active), at-risk renewals right rail, recent quotes table.

### Quotes

- **Currency** selector on quotes — USD / EUR / GBP / AUD / CAD / NZD.
- **Billing period** + **term months** on quotes — `one_time` / `monthly` / `quarterly` / `semiannual` / `annual` / `custom`. Live multi-period extrapolation shown on the totals card when recurring.
- **Per-line discount** on each line item.
- **Drag-to-reorder** line items via native HTML5 with a visual drop indicator.
- New status taxonomy in the UI: `revision`, `expired`, `archived` alongside the existing `draft` / `published` / `sent` / `accepted` / `declined`.
- Quotes list now has **filter tabs** with live counts (All / Drafts / Sent / Revision / Accepted / Declined / Expired / Archived). Archived hidden by default.
- **Currency-aware totals** everywhere (`formatCurrency(amount, currency)`).

### Quote PDF

- New **`/quotes/[id]/pdf`** print-friendly letter-sized document outside the dashboard layout. Renders org branding (logo, tagline, footer text), client info, line items with per-line discount column, totals, notes, terms & conditions. \"Print / Save as PDF\" button uses the browser's print-to-PDF.
- **\"PDF\"** button on the quote detail page opens it in a new tab.

### Branding

- New **Branding** settings tab: company name, tagline, wordmark logo URL (with preview), square mark URL (with preview), footer text, default currency.

### Categories → Phases

- Categories settings switched to a phase model: **color swatch picker**, **default phase toggle**, up/down reorder arrows, slug displayed next to name.
- One-default-per-org guaranteed at the DB level via a partial unique index.

### Coupons

- New **single_use** + **recurring** boolean flags on coupons. Surfaced as toggle switches in the create form and per-row in the table. (Enforcement at quote redemption is a follow-up.)

### Database

- **Migration 003** — `coupons.single_use`, `coupons.recurring`.
- **Migration 004** — `quotes.currency`, `quotes.discount_total`, `quotes.term_months`, `quotes.billing_period`; `quote_items.discount`; `contracts.currency`; `categories.slug` + `sort_order` + `is_default` (with one-default-per-org partial unique index); `org_settings.tagline` + `footer_text` + `mark_url` + `default_currency`.
- Migration 001 squashed to include the bootstrap RLS policies (signup chicken-and-egg fix) that were previously shipped as patch migrations 003 / 004 / 005.

### Infrastructure

- **`middleware.ts` → `proxy.ts`** for the Next 16 deprecation.
- Supabase clients accept both `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY` (the new naming) and the legacy `NEXT_PUBLIC_SUPABASE_ANON_KEY`.
- **Signup flow fix** — `supabase.auth.signUp` now fires once in the account step so retries don't hit the per-hour email rate limit on new Supabase projects.
- New **`DEPLOYMENT.md`** checklist for promoting to production (re-enable email confirmation, configure SMTP, rotate the service key, apply migrations, schedule the renewals cron).
