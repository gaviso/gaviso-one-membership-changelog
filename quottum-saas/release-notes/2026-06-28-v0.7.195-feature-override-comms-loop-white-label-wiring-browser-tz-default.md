Seven batches in one release. Closes the entire feature-override comms loop, finally wires the long-stranded white_label feature, defaults new workspaces to the user's actual timezone, and lands a handful of admin polish + security fixes.

### Operator-customer comms loop closure

- **Scheduled-change janitor fires customer email on auto-cancel paths** (\`user_matched_target\`, \`workspace_on_stripe\`) — closes the "your plan WILL switch on DATE then silently doesn't" loop.

- **Full feature-override lifecycle symmetry.** Every change to \`workspaces.feature_grants\` / \`feature_blocks\` now reaches the customer via email + auto-queued welcome popup:

  | Event | Template | Popup kind |
  |---|---|---|
  | Grant added | \`workspace.feature_granted\` (mig 084) | \`feature_grant\` |
  | Grant removed | \`workspace.feature_grant_removed\` (mig 086) | \`feature_grant_removed\` |
  | Block added | \`workspace.feature_blocked\` (mig 087) | \`feature_blocked\` |
  | Block removed | \`workspace.feature_block_removed\` (mig 087) | \`feature_block_removed\` |

  Granted features show under "What you have" on \`/dashboard/settings/billing\` with a "Granted by support" chip + any expiry; blocks show under "Temporarily unavailable".

- **Live popup push.** \`WelcomePopupPoller\` polls \`/api/me/welcome-popup/check\` every 30s (paused while tab is hidden, immediate tick on tab return) and calls \`router.refresh()\` when the workspace's \`welcome_announcement\` flips \`null → present\`. Operators pushing a popup no longer have to wait for the user to navigate — it appears within ~30s.

### UI / server-state hygiene

- **\`WelcomeAnnouncement\` calls \`router.refresh()\` on acknowledge** so any other server-rendered chunk that derives from the workspace row freshens after dismiss.

- **\`force-dynamic\` on 5 admin pages** (\`/admin/workspaces/[id]\`, \`/admin/workspaces\`, \`/admin/users\`, \`/admin/flags\`, \`/admin\`). Operator modals were already calling \`router.refresh()\` correctly — Next 16 was caching the RSC render and returning stale data. With this directive, refresh actually refreshes.

### Security

- **Closed \`client_csv_export\` bypass in unified \`/api/export\`.** The endpoint was only gating on \`data_export\`; per-entity blocks didn't enforce. Now \`entity=clients\` checks \`client_csv_export\` regardless of csv/json format. Data Settings UI shows a locked state for blocked entries.

### Feature wiring (long-overdue)

- **\`white_label\` feature finally does something.** Catalog had it since v0.7.181 but no render code referenced it — operators could grant it and the "Powered by Quottum" footer would still show everywhere. Now gated at all 5 render sites: public quote, public contract, client portal, quote PDF, contract PDF.

  Plus a **user-side toggle on \`/dashboard/settings/branding\`** — always visible (discoverable upsell), but disabled with state-specific copy:
  - \`available\` → interactive
  - \`not_on_plan\` → "Available on the Business plan. **Upgrade**"
  - \`blocked\` → "Disabled by support. Reach out if you have questions."

  Migration 085 adds \`workspace_settings.hide_quottum_branding\`. Footer hides only when BOTH the workspace has the entitlement AND the user toggled it on — entitlements grant the option, users keep agency. Default OFF so existing Business workspaces don't see their branding silently disappear.

### UX

- **New workspaces inherit the user's browser timezone** instead of defaulting to UTC. Quote dates, contract dates, reminder windows, and the dashboard timeline now render in the user's actual local zone (\`America/Los_Angeles\`, \`Europe/London\`, etc.) from the moment they sign up. Existing workspaces unchanged — operator/owner can still update via Settings → General.

### Operator polish

- **Compact icon-only remove button on feature override rows.** Was rendering as a full-width pill that looked like a primary action; now 24×24 ghost icon button on the right with a red hover state.

- **Searchable typeahead picker for the feature override Grant / Block modal.** Replaces the native \`<select>\` listing 35+ features in catalog order. Now: search by name OR key, alphabetical, existing entries grayed with a "Currently granted/blocked" tag, selected entry as a chip.

### Migrations

All four (084 + 085 + 086 + 087) already applied to prod via MCP. Files committed for the historical record.

### Smoke test path

For a workspace with a fresh white_label grant:
1. Operator grants \`white_label\` to Bobby via /admin/workspaces/[id] feature-overrides editor
2. Within 30s, popup pushes onto Bobby's screen without him touching anything
3. Bobby gets \`workspace.feature_granted\` email
4. Bobby navigates to /dashboard/settings/branding → toggle is now ENABLED (was disabled before, with "Available on Business — Upgrade")
5. Bobby checks the box, saves
6. Bobby opens a public quote URL → "Powered by Quottum" footer is gone
7. Operator removes the grant via the × button → Bobby gets a second popup + email ("\`white_label\` was removed")
8. Toggle on /dashboard/settings/branding flips back to disabled
