Two improvements bundled.

## 1. Plan renames now flow through to Stripe price nicknames

Previously: renaming a plan in Quottum (e.g. Pro → Deluxe) updated the Stripe Product name on next sync, but left existing prices with stale nicknames (`Quottum Pro — monthly` still showed in Stripe after the rename to Deluxe). Stripe price IDs are intentionally not rotated when only the name changes — subscribers stay attached — but the cosmetic nickname should follow.

Fix: when the sync sees an existing price with the right amount but a stale nickname, it calls `stripe.prices.update(id, { nickname })` to refresh just that field. Price ID, amount, and subscriber attachments are untouched.

## 2. "Which option to pick" on the Stripe API key chooser

Stripe's "Create a secret key" dialog opens with a 3-option chooser that's easy to get wrong:
- ✅ **Powering an integration you built** — for Quottum (your server is the integration)
- ❌ Providing this key to a third-party application — for plugins (Zapier, n8n)
- ❌ Authorizing an AI agent — for Claude/Cursor MCP access

New `STRIPE_KEY_OPTION` constant in [`src/lib/stripe-webhook-events.ts`](https://github.com/gaviso/quottum-saas/blob/v0.7.171/src/lib/stripe-webhook-events.ts) is the canonical source. All three setup surfaces now render the guidance:
