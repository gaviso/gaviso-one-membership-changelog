First slice of the operator subscription-actions surface on \`/admin/workspaces/[id]\`. Beta-gated behind \`operator_subscription_actions_v179\` (the same key will gate the v0.7.182 follow-ups — the underlying capability is one feature).

### Added

- **Terminate workspace** action on \`/admin/workspaces/[id]\` (danger zone Card). Opens a modal with an always-visible yellow policy banner pointing customer-requested cancellation to **Cancel Subscription** instead.
  - **Reason taxonomy (G)** — \`policy_violation\`, \`fraud_cleanup\`, \`beta_graduation\`, \`complementary_partner\`, \`support_compensation\`, \`goodwill\`, \`migration_from_competitor\`, \`customer_request\`, \`other\`.
  - **Optional operator notes** — free-text, internal only, never reaches the customer.
  - **Stripe** — auto-cancels any active subscription with \`prorate: false, invoice_now: false\` (**no credit**, per policy B). Stale / missing sub_ids are treated as soft-success.
  - **Side effects** — sets \`workspaces.terminated_at\` + reason fields, sets \`cancel_at_period_end=true\` defensively, cancels any pending \`workspace_plan_changes\` with reason \`workspace_terminated\`.
- **\`billing.workspace_terminated\`** email — fires on termination; ships the reason summary (mapped from the G taxonomy), never the raw operator notes.
- **TerminatedBanner** — full-width red banner at the top of the dashboard when the current workspace is terminated. Read + export still work.
- **Migration 077** — additive schema for both v0.7.181 *and* v0.7.182:
  - \`workspaces.terminated_at\`, \`termination_reason_category/_text\`, \`feature_grants/feature_blocks\` (v0.7.182), \`welcome_announcement\` (v0.7.182)
  - \`workspace_plan_changes\` table + RLS + cron-friendly partial index (v0.7.182)
  - Three email templates seeded: \`billing.workspace_terminated\`, \`billing.plan_changed\`, \`billing.scheduled_plan_change\`
  - \`beta_features\` row \`operator_subscription_actions_v179\` granted to all operators

### Write-gated paths

Server-side 423 Locked (\`code='workspace_terminated'\`) when terminated:

- \`POST/PATCH/DELETE /api/quotes(/[id])\`
- \`POST/PATCH/DELETE /api/contracts(/[id])\`
- \`POST/PATCH/DELETE /api/clients(/[id])\`
- \`PATCH /api/settings\`
- \`PATCH /api/workspaces/[id]\`

Other workspace-scoped writes (line-items, coupons, members, beta feedback, etc.) are **not** gated yet — exhaustive coverage moves to RLS in v0.7.182.
