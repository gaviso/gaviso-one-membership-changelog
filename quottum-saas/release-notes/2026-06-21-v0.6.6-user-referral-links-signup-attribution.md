Every user now has a referral link. Share it, friends sign up through it, you see them in your dashboard. No rewards system yet — just the plumbing + visibility so we can layer rewards on later without re-architecting.

### What you get

New sidebar entry: **Workspace → Refer**. Page shows:

- Your shareable link (\`https://app.quottum.io/signup?ref=XXXXXXXX\`) with a one-click copy button + native share sheet on mobile
- Count of people you've referred
- Table of referred signups with email + date

### Schema (migration 020)

\`\`\`
referral_codes (user_id pk, code unique)
referrals      (referrer_user_id, referee_user_id, code, signed_up_at)
                unique(referee_user_id) — can only be referred once
\`\`\`

Codes are 8-char strings from a no-ambiguity alphabet (no 0/O, 1/I/L) so they're safe to share verbally or in chat. ~8.5×10¹¹ keyspace before any collision risk.

### Signup integration

\`/signup?ref=ABCD1234\` →

1. Code stashed in \`sessionStorage\` (survives reloads + the full signup round-trip)
2. Visible \"You're signing up via a referral · ABCD1234\" banner so the new user knows it's working
3. After user + personal workspace creation, fires a fire-and-forget \`POST /api/me/referral { action:'record' }\` — never blocks the redirect
4. Unknown codes silently no-op server-side (don't leak code validity to the signup form)
5. Self-referral filtered (can't refer yourself)

### What's NOT in this release

- **No rewards yet** — referrer and referee both get nothing beyond attribution. Foundation for adding rewards (extended trial, plan credits, etc.) without schema changes — just trigger off the \`referrals\` table.
- **No public leaderboard** — referrals are only visible to the people involved (RLS enforces this).
- **No email notification when someone signs up via your link** — could add later via the \`sendEmail()\` helper.
