Closes the biggest behavioural gap vs the Quottum WP plugin: contracts now have the full lifecycle of statuses, lineage, and actions. **Phase 1 of the v0.3 → v1.0 roadmap.**

### Status taxonomy

Contracts now support six statuses (was three):

- **Active** · **Renewed** · **Expired** · **Terminated** · **Archived** · Cancelled

Filter tabs on the contracts list with live counts; archived hidden by default. Status chips throughout the app pick up the new tones.

### Actions

- **Terminate** with an optional reason; the reason is captured in the activity timeline.
- **Archive** to drop a contract out of the working set; **Restore** brings it back.
- **Renew** clones the contract with a new term — pre-filled from the source, client locked, start date constrained to ≥ source.end_date, end date auto-derived from start + term. Source contract is marked `renewed`; both rows link to each other via `renewed_from_id` / `renewed_into_id`.
- Status-aware buttons in the new ContractActions toolbar.

### Detail page rebuild

Four new panels alongside the existing activity timeline:

1. **KPI strip** — Total Contract Value, MRR, ARR, Days remaining.
2. **Lifecycle timeline** — 5-node strip (Start · 25% · Midpoint · Renewal window · End) for recurring; 2-node for one-time. Elapsed-time fill bar.
3. **Renewal history** — walks the `renewed_from_id ↔ renewed_into_id` chain in both directions, current term highlighted.
4. **Reminder schedule** — reads `org_settings.renewal_milestones` × `renewal_log` to show *fired / due / upcoming / missed / skipped* per milestone.

### Contracts list rebuild

- Status filter tabs with counts (pill-seg pattern).
- Text search across number / title / client name.
- Sortable headers (number, amount, MRR, end date).
- Days-remaining chip when the contract is active.

### Schema (migration 006)

```
contracts:
  + archived_at          timestamptz
  + terminated_at        timestamptz
  + terminated_reason    text
  + renewed_from_id      uuid references contracts(id)
  + renewed_into_id      uuid references contracts(id)
+ index on renewed_from_id
+ index on archived_at
```

`ContractStatus` widened to `active | renewed | expired | terminated | archived | cancelled`. `BillingCycle` widened to add `semiannual` + `custom`. No DB CHECK constraints — code-side validation, same convention as quotes.

### API

- `PATCH /api/contracts/[id]` now takes an `action` verb: `terminate` (with optional reason) · `archive` · `restore`. Each action writes a `contract_events` row.
- `POST /api/contracts/[id]/renew` — clones with new term, links lineage, marks source as renewed, refuses double-renewal.

### Misc

- Settings index page redirects to `/branding` (was `/company`) so clicking **Settings** in the sidebar lands somewhere useful.
