## Beta signup — structured questionnaire
Three new questions on `/beta-signup` so beta intake is comparable to the WordPress Lite waitlist:

- **What best describes you?** — Freelancer / Agency / In-house at a business / Developer / Other *(required)*
- **What's the #1 thing you wish your current solution did?** — free text *(optional)*
- **How soon do you need this?** — Now / In the next 3 months / Just exploring *(required)*

The existing "What you'll mostly use it for" + "What do you currently use?" + "How did you hear?" fields all stay.

## Operator console — filter & at-a-glance
On `/admin/beta`:

- **Filter bar** — role + urgency selects appear once at least one tester has structured intake. Counts show *"3 of 12"* so it's clear what's hidden.
- **Tester cards** — three chips per card (role · primary use · urgency). Urgency = *"Now"* renders warn-toned so urgent testers visually pop.

## Schema
`beta_testers` gained `role`, `primary_use`, `top_wish`, `urgency` columns with the same check constraints as `wp_lite_waitlist` — so a single SQL query can compare both audiences (*"all freelancers with urgency=now, across both sources"*).
